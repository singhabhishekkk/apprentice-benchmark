# paperless-gpt document type classification

[paperless-gpt](https://github.com/icereed/paperless-gpt) is an MIT-licensed companion for paperless-ngx. It uses LLMs over OCR text to suggest metadata such as title, correspondent, tags, and document type.

## Why this slice

Document type is a bounded classification task with a real production prompt and a public gold dataset. The shipped paperless-gpt prompt already asks for one type name from an allowed list, so the benchmark can measure the prompt as users run it instead of inventing a synthetic task.

This task uses OCR text from a corrected Tobacco3482 variant on Hugging Face. The labels are real document-type labels, not generated by a model.

## Projected cost per 1,000 documents

OpenAI's published GPT-5.4 mini text pricing is $0.75 per 1M input tokens and $4.50 per 1M output tokens. For 1,000 document-type calls:

```text
projected_cost_usd =
  (input_tokens_per_doc * 1000 / 1_000_000 * 0.75)
  + (output_tokens_per_doc * 1000 / 1_000_000 * 4.50)
```

The tokenizer-specific token count is pending a real benchmark run. Until that run prints token totals, this page publishes the formula and published rates only.

## Local-serving angle

paperless-gpt already supports Ollama. A fine-tuned 4B classifier can be served locally behind the same kind of endpoint for this one-field decision, keeping document text on the user's machine while preserving the allowed-type output contract.

## Caveats

This benchmark covers document type only. Real paperless-gpt installs may use different type lists, OCR quality, document languages, and longer workflows that also generate title, correspondent, and tags.
